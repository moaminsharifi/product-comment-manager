Introduction

api documentation with various version

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

127.0.0.1:8080

Authenticating requests

This API is not authenticated.

Endpoints

Login User.

This endpoint Log user and return token.

Example request:
curl --request POST \
    "127.0.0.1:8080/api/auth/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"louie22@example.com\",
    \"password\": \"qvcrjhpzjazrhctlqovvhwmxyhyqjjfihlbuebjhdcwdsjfdaezschmluazadvixqkaznaoyiymnawksvhkmvqdqhopuyqiycubyifxbjnhtwjkvsuqxishanhxalyc\"
}"

const url = new URL(
    "127.0.0.1:8080/api/auth/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "louie22@example.com",
    "password": "qvcrjhpzjazrhctlqovvhwmxyhyqjjfihlbuebjhdcwdsjfdaezschmluazadvixqkaznaoyiymnawksvhkmvqdqhopuyqiycubyifxbjnhtwjkvsuqxishanhxalyc"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200):


{
    "data": {
        "id": 11,
        "name": "laravel",
        "email": "test@local.com",
        "token": "10|dlzWqb87FWA3adXJb0Rzmj471CMebyWKnj7jJkIC",
        "token_type": "Bearer"
    }
}

Example response (422, Invalid Email):


{
    "message": "The selected email is invalid.",
    "errors": {
        "email": [
            "The selected email is invalid."
        ]
    }
}

Example response (422, Invalid Password):


{
    "error": {
        "code": "10002",
        "message": "Password Not Correct"
    },
    "data": []
}

Request      

POST api/auth/signup

Body Parameters

name  string  

Must not be greater than 180 characters. Must be at least 3 characters.

email  string  

Must be a valid email address. Must not be greater than 180 characters.

password  string  

Must not be greater than 180 characters. Must be at least 6 characters.

password_confirm  string  

The value and password must match. Must not be greater than 180 characters.

Store Comment.

requires authentication

This endpoint Add new comment to system. If product not exist then create anonymous product.

Example request:
curl --request POST \
    "127.0.0.1:8080/api/comments" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"productName\": \"\",
    \"comment\": \"b\"
}"

const url = new URL(
    "127.0.0.1:8080/api/comments"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "productName": "",
    "comment": "b"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200):


{
    "data": {
        "id": 2,
        "comment": "comment"
    }
}

Example response (422, Invalid inputs):


{
    "data": {
        "id": 2,
        "comment": "comment"
    }
}
